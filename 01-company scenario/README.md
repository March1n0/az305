# az305
Project on how a business moves from on premisses to Azure cloud

## Overview

This project was created to design and implement a cloud architecture solution in Microsoft Azure based on a business scenario involving:

- 150 employees
- One head office
- Remote workers
- On-premises Active Directory
- File server
- SQL database
- Public-facing website

The goal of this project was to apply Azure Cloud Architecture principles, governance, networking, identity, security, and workload design concepts in a practical environment.

---

## Project Limitation

This project was completed using a **Microsoft Azure Free Account**. As a result, certain services, features, and enterprise-level configurations could not be fully implemented due to subscription, licensing, and cost limitations.

Examples of potential limitations include:

- Microsoft Entra ID Premium (P1/P2) features
- Conditional Access Policies
- Privileged Identity Management (PIM)
- Enterprise-scale production environments
- Certain Azure services that require paid subscriptions
- Production-grade disaster recovery configurations
- Advanced security and governance features

---

## How Missing Components Were Addressed

Where implementation was not possible due to free-tier restrictions, I have included:

- Detailed architectural designs
- Recommended Azure services
- Configuration guidance
- Implementation steps
- Design decisions and justifications

This ensures that the project demonstrates an understanding of how the solution would be implemented in a real-world enterprise environment, even when the service could not be deployed directly.

---

## Architecture Approach

The project follows a cloud architecture methodology that includes:

1. Requirements Gathering
2. Landing Zone Design
3. Identity and Access Management
4. Networking Design
5. Compute and Storage Design
6. Database Design
7. Security and Governance
8. Monitoring and Logging
9. Backup and Disaster Recovery

---

## Objective

The objective of this project is not only to deploy Azure resources, but also to demonstrate the planning, design, governance, security, and architectural decision-making expected of a Cloud Architect.

Where free-tier limitations prevented implementation, supporting documentation has been provided to explain:

- What service should be used
- Why it should be used
- How it would be implemented in a production environment
- Any licensing or subscription requirements

---

## Disclaimer

This project is intended for learning, portfolio, and cloud architecture demonstration purposes. Some components have been simulated, documented, or represented through architectural designs due to the limitations of a free Azure subscription.

All design recommendations are based on Microsoft Azure best practices and enterprise cloud architecture principles.
